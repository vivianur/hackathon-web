### HACKATHON

Sobre o Hackaton:
O Hackaton é o projeto final da Pós Graduação, sendo o momento em
que você irá colocar em prática tudo que foi absorvido em cada fase do curso. O
briefing abaixo é o guia do seu projeto, colocados os requisitos e tecnologias que
esperamos que você utilize.

O prêmio do Hackathon é dado ao grupo que melhor desenvolver o
projeto, usando a criatividade, inovação e completude. Ou seja, além das
features e requisitos abaixo, sinta-se livre para inovar, sugerir e implementar
outras features para além do que foi proposto! Todos os grupos que entregarem
o hackaton recebem nota pelo projeto.

#### BRIEFING – TEMA: ACESSIBILIDADE COGNITIVA

A instituição FIAP Inclusive está iniciando um movimento para melhorar
a experiência digital de seus estudantes e colaboradores com dificuldades
cognitivas, como:

```
● TDAH;
● TEA (Autismo);
● Dislexia;
● Burnout e sobrecarga mental;
● Dificuldades de foco e retenção;
● Ansiedade em ambientes digitais;
● Sobrecarga sensorial.
```
Esses usuários relataram grande dificuldade ao utilizar plataformas atuais
por causa de:

```
● Excesso de informação na tela;
● Falta de previsibilidade e consistência;
● Ausência de modo de foco;
● Textos longos e pouco adaptados;
● Navegação caótica;
```

```
● Ausência de recursos de organização cognitiva.
```
Para resolver esse problema, a FIAP Inclusive decidiu desenvolver uma
plataforma chamada MindEase, cujo objetivo é facilitar a vida acadêmica e
profissional de pessoas neurodivergentes e/ou com desafios de processamento
cognitivo.

Você e seu grupo foram contratados para desenvolver a versão Web e a
versão Mobile, aplicando tudo que aprenderam em:

```
● Arquitetura (Clean, Microfrontends, Componentização);
● Desenvolvimento Web (React, Angular, Next.js);
● Desenvolvimento Mobile (Flutter / React Native);
● Acessibilidade digital;
● Testes, CI/CD e boas práticas.
```
O Projeto MindEase deve conter

```
Painel Cognitivo Personalizável
Um dashboard onde o usuário ajusta:
● Nível de complexidade da interface;
● Modo de foco (esconde distrações);
● Modo resumo / modo detalhado;
● Contraste, espaçamento e tamanho de fonte;
● Alertas cognitivos (ex.: “você está muito tempo nesta tarefa”).
```
```
Organizador de Tarefas com Suporte Cognitivo
Um sistema de tarefas com:
● Etapas visuais (Kanban simplificado);
● Timers com controle de foco (método Pomodoro adaptado);
● Checklist inteligente para reduzir sobrecarga;
● Avisos de transição suave entre atividades.
```

```
Perfil do Usuário + Configurações Persistentes
Armazenar preferências como:
● Modo de foco;
● Intensidade de contraste e espaçamento;
● Perfil de navegação;
● Necessidades específicas;
● Rotinas de estudo ou trabalho.
```
Tecnologias Esperadas

```
Arquitetura Microfrontend
● Separação clara entre módulos (painel, biblioteca, tarefas, perfil);
● Comunicação entre microapps.
```
```
Desenvolvimento Mobile
● Flutter ou React Native;
● Versão mobile deve manter coerência cognitiva com a versão Web.
```
```
Desenvolvimento Web
● Typescript;
● Angular, React ou Next.js.
```
```
Clean Architecture
● Camada de domínio isolada;
● Casos de uso independentes de UI;
● Adaptadores e interfaces claras.
```
```
Acessibilidade Cognitiva (obrigatório)
● Níveis ajustáveis de complexidade;
● Componentes de foco;
● Ritmos guiados na interface;
● Redução de estímulos visuais;
● Animações controláveis.
```
```
Esse é o diferencial temático do hackathon.
```
